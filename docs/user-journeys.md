# Afribac User Journeys (Draft)

Purpose: capture the primary flows for each role to drive UX specs, scope, and priorities.

## Student journey
### Goals
- Find relevant content fast (country, series, subject, topic).
- Study consistently, track progress, and improve weak areas.
- Practice under exam conditions and review corrections.
- Stay motivated through rewards, streaks, and peer support.

### Entry triggers
- First-time signup and onboarding.
- Returning to continue a course or practice.
- Reminders for daily plan or missed revision queue.

### Happy path
1. Sign up and complete onboarding (name, country, series, goals).
2. Land on dashboard with personalized next steps.
3. Pick a course and start learning (content, PDF, video).
4. Take a quiz/exercise and review results.
5. Start an exam simulation and submit answers.
6. Review corrections and update mastery map.
7. Follow the study plan and spaced repetition queue.

### Key moments
- First course completion (reward and motivation boost).
- First exam simulation with clear summary of gaps.
- Visible improvement in mastery map or predicted score.

### Edge cases
- Low bandwidth or device storage limits.
- Long gaps in activity (streak recovery).
- Content not available for a specific country/series.

### Success metrics
- Activation rate (first course started within 24h).
- Weekly active learners and time on task.
- Quiz accuracy trend and exam simulation completion rate.
- Retention across the prep season.

### Data and feature dependencies
- Courses, quizzes, exams, progress, attempts, mastery, planner, badges.

## Member journey
### Goals
- Create high-quality content quickly and publish with confidence.
- Track impact and improve based on usage data.

### Entry triggers
- Member onboarding and content creation access.
- Requests for new topics or curriculum updates.

### Happy path
1. Enter content workspace with country scope.
2. Create or edit a course, quiz, or exam with media.
3. Submit for review or publish (based on workflow).
4. Monitor analytics: views, completion, feedback.
5. Improve content and update as needed.

### Edge cases
- Missing assets or poor formatting in uploaded PDFs.
- Inconsistent curriculum mapping.

### Success metrics
- Content publish velocity and quality score.
- Student engagement and completion rates.

### Data and feature dependencies
- Content tables, review workflow, assets, analytics, activity logs.

## Admin journey
### Goals
- Keep the platform safe, consistent, and aligned to curricula.
- Monitor engagement and growth across countries.

### Entry triggers
- New country or series onboarding.
- Reported content or user behavior.

### Happy path
1. Configure countries, series, and subjects.
2. Manage users (roles, status, access).
3. Review platform analytics and activity logs.
4. Moderate reported content and resolve issues.
5. Support partners and educators with insights.

### Edge cases
- High activity spikes near exam periods.
- Content disputes or copyright issues.

### Success metrics
- Content coverage per country/series.
- User safety response times.
- Platform uptime during exam peaks.

### Data and feature dependencies
- Admin config tables, user management, analytics, moderation queue.

## Parent/Guardian (optional)
### Goals
- Monitor progress, encourage consistency, and support outcomes.

### Happy path
1. Invite or link to student account.
2. View progress summary and weak areas.
3. Receive reminders for upcoming deadlines.
